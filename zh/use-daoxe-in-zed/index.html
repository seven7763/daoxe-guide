在 Zed 里用 DaoXE

Zed 的 Agent 支持自定义 OpenAI 兼容供应商。在 language_models.openai_compatible 下加入 DaoXE，就能在一个快速的原生编辑器里用 GPT、Claude、Gemini、DeepSeek 等 —— 一个 key。

更新于 2026-07-20

Zed 允许在 Agent 设置界面或直接在 settings.json 里注册 OpenAI 兼容供应商。把 api_url 指向 DaoXE 并列出你要的模型；key 不放在文件里（用环境变量或 UI）。一个 key 覆盖各家 —— 见最便宜 API 教程。

为什么让 Zed 走 DaoXE#

  • 原生、快速。 Zed 的 Agent 面板用你的 DaoXE 模型做编辑。
  • 显式声明模型。 在 available_models 里写准确 ID 和上下文窗口。
  • 一个 key，多家厂商。 无需分别开 GPT/Claude/Gemini/DeepSeek 账号。

在 Zed 里配置#

  1. 打开 Agent 设置。 执行 agent: open settings，在 LLM Providers 点 Add Provider —— 或直接编辑 settings.json。
  2. 设 api_url 和模型。 在 language_models.openai_compatible 下加一个供应商：api_url: https://daoxe.com/v1，并在 available_models 写一项（准确 ID + max_tokens）。
  3. 安全地提供 key。 在 UI 里填，或设环境变量 DAOXE_API_KEY —— Zed 会按供应商名推导变量名。
  4. 选择并测试。 在 Agent 面板选中该模型，做一次小编辑确认可用。
json
// Zed settings.json  (agent: open settings -> Add Provider, or edit directly)
{
  "language_models": {
    "openai_compatible": {
      "DaoXE": {
        "api_url": "https://daoxe.com/v1",
        "available_models": [
          { "name": "YOUR_EXACT_MODEL_ID", "display_name": "DaoXE model", "max_tokens": 128000 }
        ]
      }
    }
  }
}
// Key is NOT stored here. Set it in the UI, or as env DAOXE_API_KEY
// (Zed derives <PROVIDER_ID_UPPERCASE>_API_KEY from the provider name).

值得注意的坑

切勿把 key 写进 settings.json —— Zed 读取名为 <供应商ID大写>_API_KEY 的环境变量，所以供应商 DaoXE 对应 DAOXE_API_KEY。api_url 必须含 /v1。推理模型需设 reasoning_effort 和对应 capabilities。ID 与账号绑定 —— 见 GET /v1/models。

验证你拿到的是真模型#

先证明端点可用，再怀疑客户端 —— 这一步失败，改任何设置都没用：

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

先脱离 Zed 独立验证端点，再在 temperature 0 下把固定 prompt 与官方 API 对比：

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

常见问题#

Zed 如何保存我的 key？

不放在 settings.json。用 UI，或设 DAOXE_API_KEY —— Zed 按供应商名推导变量名（大写 + _API_KEY）。

api_url 填什么？

https://daoxe.com/v1，必须含 /v1。

一定要列出模型吗？

要 —— 在 available_models 里加准确 ID 和 max_tokens（上下文窗口）。

推理模型不思考？

在该模型项里设 reasoning_effort 和相应 capabilities。

试用 DaoXE —— 并亲自 benchmark 它
