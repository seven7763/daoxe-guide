在 opencode 里用 DaoXE

opencode 是终端 AI 编码智能体，供应商系统可插拔。在 opencode.json 里把 DaoXE 加为自定义供应商，即可用 GPT、Claude、Gemini、DeepSeek 等来驱动它 —— 一个 key。

更新于 2026-07-20

opencode 在 opencode.json 里用 AI-SDK 适配器配置供应商。用 @ai-sdk/openai-compatible，把 baseURL 指向 DaoXE，并列出模型 ID。一个供应商块就解锁 key 背后的各家 —— 见最便宜 API 教程。

为什么让 opencode 走 DaoXE#

  • 终端智能体。 opencode 在 shell 里用你的 DaoXE 模型执行多步任务。
  • 声明式供应商。 一段 JSON 就加上 DaoXE 及其模型。
  • 一个 key，多家厂商。 GPT、Claude、Gemini、DeepSeek、Kimi 都在一个 key 后面。

在 opencode 里配置#

  1. 打开 opencode.json。 创建/编辑 opencode.json（项目）或 ~/.config/opencode/opencode.json（全局）。
  2. 添加供应商。 在 provider 下用 npm: "@ai-sdk/openai-compatible"，设 options.baseURL 为 https://daoxe.com/v1。
  3. 列出准确模型 ID。 在 models 下加与 GET /v1/models 一致的键。key 通过 options.apiKey 引用环境变量。
  4. 重启并验证。 重启 opencode，运行 /models 确认 DaoXE 及其模型出现。
json
// opencode.json (project) or ~/.config/opencode/opencode.json (global)
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "daoxe": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "DaoXE",
      "options": {
        "baseURL": "https://daoxe.com/v1",
        "apiKey": "{env:DAOXE_API_KEY}"
      },
      "models": {
        "YOUR_EXACT_MODEL_ID": { "name": "DaoXE model" }
      }
    }
  }
}

值得注意的坑

用 @ai-sdk/openai-compatible 走 /v1/chat/completions；若某模型需要 /v1/responses，则该模型改用 @ai-sdk/openai。模型键必须与账号 ID 完全一致（GET /v1/models）。可从 models.dev 拉取限额/价格元数据，但模型键仍须是真实 ID。改完配置要重启。

验证你拿到的是真模型#

先证明端点可用，再怀疑客户端 —— 这一步失败，改任何设置都没用：

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

先独立验证端点，再在 temperature 0 下把固定 prompt 与官方 API 对比：

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

常见问题#

该用哪个 npm 适配器？

chat completions 用 @ai-sdk/openai-compatible。需要 /v1/responses 的模型改用 @ai-sdk/openai。

base URL 放哪？

供应商块里的 options.baseURL = https://daoxe.com/v1。

模型没出现在 /models？

模型键必须等于 GET /v1/models 里的准确 ID，且改配置后需重启 opencode。

怎么让 key 不进文件？

在 options.apiKey 引用环境变量，并导出 DAOXE_API_KEY。

试用 DaoXE —— 并亲自 benchmark 它

一个 key 调 GPT、Claude、Gemini、DeepSeek 等。用开源 benchmark 对准我们做对比 —— 别只听我们说。
