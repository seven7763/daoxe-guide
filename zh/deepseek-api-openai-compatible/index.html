DeepSeek API 的 OpenAI 兼容接入 —— 以及 reasoning_content 这个坑

DeepSeek 便宜、推理强。通过 DaoXE 就是一次普通的 OpenAI 兼容调用 —— 但 thinking 版本有个真实的多轮坑，上线前值得知道。

更新于 2026-07-20

DeepSeek 是性价比最高的家族之一，而在海外注册常常很痛苦 —— 通过 DaoXE，它和 GPT、Claude 在同一个 key 后面。用 deepseek-* ID 调 /v1/chat/completions。如何做便宜/顶级分工，见最便宜 API 教程。

通过 DaoXE 调用 DeepSeek#

用 /v1/chat/completions，模型填准确的 deepseek-* ID。有非思考版与思考版；思考版会返回单独的 reasoning_content 字段。

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'
python
# DeepSeek "thinking" variants return a separate reasoning_content field.
# In multi-turn tool use you MUST echo the previous assistant message back
# (reasoning_content included) or the API answers 400.
from openai import OpenAI

client = OpenAI(base_url="https://daoxe.com/v1", api_key="YOUR_DAOXE_KEY")
r = client.chat.completions.create(
    model="YOUR_EXACT_MODEL_ID",   # a deepseek-* id from GET /v1/models
    messages=[{"role": "user", "content": "2+2? Think briefly, then answer."}],
)
msg = r.choices[0].message
print(getattr(msg, "reasoning_content", None))  # chain-of-thought, if exposed
print(msg.content)                               # final answer

DeepSeek 特有的注意点#

  • reasoning_content 回传。 在多轮工具调用里用思考版时，必须把上一条 assistant 消息（含 reasoning_content）回传，否则 API 返回 400。
  • 非思考版无需回传。 若不需要显式推理，用非思考版即可完全避开这个坑。
  • 框架可能丢字段。 有些客户端会静默丢弃 reasoning_content —— 开思考前先检查你的客户端。
  • 只用准确 ID。 从 GET /v1/models 按 ID 选对版本。

常见问题#

为什么第二轮报 400？

思考版要求把上一条 assistant 消息连同 reasoning_content 回传到下一次请求；缺了就返回 400。

怎么避开 reasoning_content 坑？

要么忠实回传 reasoning_content，要么在不需要显式推理时用非思考版。

海外没有中国账号能用 DeepSeek 吗？

能 —— 通过 DaoXE 它在你的单一 key 后面，免去常见的海外注册摩擦。

这里 DeepSeek 多少钱？

按模型、按账号 —— 见实时定价。相对官方优势：多数主流模型经低价分组约低 30%–80%。
