在 Cherry Studio 里用 DaoXE

Cherry Studio 是跨平台桌面客户端，支持众多供应商。把 DaoXE 加为 OpenAI 类型供应商，就能用一个 key 调 GPT、Claude、Gemini、DeepSeek 等 —— 前提是把 API 地址的路径规则填对。

更新于 2026-07-20

Cherry Studio 把 DaoXE 当作 OpenAI 类型供应商：粘贴 key、填 API 地址、加模型 ID。唯一容易踩坑的是 Cherry 如何补全端点路径 —— 填对就一切正常。便宜/顶级模型分工见最便宜 API 教程。

为什么让 Cherry Studio 走 DaoXE#

  • 本地桌面，多家供应商。 聊天数据留在本地，模型由 DaoXE 提供。
  • 一个 key，多种模型。 在一个 DaoXE 供应商下加入 GPT、Claude、Gemini、DeepSeek 的 ID。
  • 按助手配模型。 给不同助手配不同 DaoXE 模型，平衡成本与质量。

在 Cherry Studio 里配置#

  1. 添加供应商。 设置 → 模型服务 → 添加，类型选 OpenAI，命名为 DaoXE。
  2. 填 key 与 API 地址。 粘贴 DaoXE key。地址填 https://daoxe.com，让 Cherry 自动补路径（准确规则见注意点）。
  3. 添加模型。 从 GET /v1/models 拉取或手动添加准确模型 ID（如一个 GPT、一个 Claude、一个 DeepSeek 的 ID）。
  4. 选模型并测试。 在对话里选一个 DaoXE 模型，发条消息确认能正常响应。

值得注意的坑

经典坑是 Cherry 的 API 地址路径补全。可靠做法：填 https://daoxe.com 让 Cherry 补上 /v1/chat/completions；或填完整端点并以 # 结尾，强制原样使用。以 / 结尾又是另一种处理方式 —— 若模型列表加载不出或报 404，几乎都是这条规则。用 GET /v1/models 的准确 ID；若自动拉取结果不同就手动添加。key 存在本地应用里。

验证你拿到的是真模型#

先证明端点可用，再怀疑客户端 —— 这一步失败，改任何设置都没用：

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

先在 Cherry 之外独立验证端点，再在 temperature 0 下把固定 prompt 与官方 API 对比：

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

常见问题#

该选哪种供应商类型？

选 OpenAI —— DaoXE 是 OpenAI 兼容的。填 key 和 API 地址，再加模型 ID。

API 地址该填什么？

填 https://daoxe.com 让 Cherry 补 /v1/chat/completions；或填完整路径并以 # 结尾强制原样。

模型加载不出或 404？

几乎都是 API 地址路径规则。按注意点重填地址，并用 GET /v1/models 确认 key 有效。

不同助手能用不同模型吗？

能 —— 每个助手配一个 DaoXE 模型，全在一个 key 下。当前价格优势：多数主流模型经低价分组约低 30%–80%。
