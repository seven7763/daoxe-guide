在 Cline 里用 DaoXE

Cline 是 VS Code 里的自主编码智能体。把它指向 DaoXE 的 OpenAI 兼容端点，就能用一个 key 驱动它用 GPT、Claude、Gemini、DeepSeek 等做文件编辑和工具调用。

更新于 2026-07-20

Cline 支持任何 OpenAI 兼容端点，所以切到 DaoXE 只是改 base URL 和 key —— 无需代理、无需魔改。好处：一个 key 覆盖各家的最便宜 API 教程，让智能体用顶级模型规划、用便宜模型编辑。

为什么让 Cline 走 DaoXE#

  • 一个 key，多种模型。 切换智能体模型，无需在各家分别开户、绑卡、对账。
  • 按成本分流。 日常编辑用小模型，规划用顶级模型，全在一个余额里。
  • 原生工具调用。 只要模型支持工具，Cline 的自主编辑与命令执行就能正常工作。

在 Cline 里配置#

  1. 打开供应商设置。 在 Cline 面板点齿轮，把 API Provider 设为 OpenAI Compatible。
  2. 填 Base URL。 填 https://daoxe.com/v1 —— 到 /v1 为止，不要带 /chat/completions（Cline 会自己补路径）。
  3. 粘贴 key 和 Model ID。 填入 DaoXE key，再填一个从 GET /v1/models 原样复制的 Model ID。乱猜的名字会失败。
  4. 点 Verify。 Cline 会发一次真实请求；通过后跑个小任务，确认编辑和工具调用正常。

值得注意的坑

Base URL 必须到 /v1 为止 —— 末尾带 /chat/completions 会拼出错误 URL。模型 ID 与账号绑定，请从 GET /v1/models 读取。智能体模式需要真正支持工具调用的模型；若编辑不生效，换个支持工具的模型并在 Model Configuration 里设好上下文窗口。更多客户端见客户端设置说明。

验证你拿到的是真模型#

先证明端点可用，再怀疑客户端 —— 这一步失败，改任何设置都没用：

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

便宜端点只有在给你真模型时才算省钱。先证明连通，再在 temperature 0 下把输出与官方 API 对比：

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

常见问题#

Cline 能用 DaoXE 吗？

能 —— 选 OpenAI Compatible 供应商，base URL 填 https://daoxe.com/v1，再填 key 和准确模型 ID。

为什么 Model ID 提示 not found？

该 ID 必须与你账号 GET /v1/models 里的字符串一致。请原样复制；ID 与账号绑定。

同一套配置能同时用 Claude 和 GPT 吗？

能 —— 添加多个模型 ID，在一个余额下切换。当前价格优势：多数主流模型经低价分组约低 30%–80%。

智能体编辑不生效？

用支持原生工具调用的模型，并在 Model Configuration 里设置足够大的上下文窗口。
