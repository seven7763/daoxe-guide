OpenAI 兼容 · 多协议 · 面向出海开发者

一个 key，调用所有主流大模型。可被 benchmark 验证。

DaoXE 是一个 OpenAI 兼容、多协议的 LLM API 网关。把 Cursor、Claude Code、Cline 或你自己的代码指向一个 base URL，即可在 GPT、Claude、Gemini、DeepSeek 等之间切换 —— 并用开源 benchmark 自行验证拿到的是真模型。

OpenAI 兼容 · 原生 Anthropic Messages · 开源可复现 benchmark

直接对接多家供应商意味着分别开号、绑卡、对账，而且在不少地区官方支付常常失败。网关把这些收敛成 一个 base URL（https://daoxe.com/v1）、一个 key、一份账单，覆盖 GPT · Claude · Gemini · DeepSeek · Kimi · Qwen · Doubao。本教程讲怎么把 DaoXE 用好、用得诚实，以及如何验证。

为什么用网关 —— 以及为什么是它#

一个 key，多种模型

GPT、Claude、Gemini、DeepSeek、Kimi 等统一在一个 endpoint、一份账单下。不用逐家开号、绑卡、对账。

三种协议，原生支持

OpenAI Chat Completions、OpenAI Responses，以及 Anthropic Messages —— Claude Code 能正常跑，而不是套壳。

诚实不掉包

不搞悄悄降级。账户维度的模型列表、透明用量，以及一套能抓降级的方法。

可验证，而非靠信任

把开源 benchmark对准 DaoXE 和任何一家。结果可复现 —— 你可以自己重跑。

无缝接入你的工具

在 Cursor、Cline、Aider、LobeChat 以及 OpenAI/Anthropic SDK 里改一个 base URL 即可，旧代码照常工作。

为出海开发者而建

面向出海开发者、海外华人与俄语开发者。不向中国大陆提供服务。

三步开始#

  1. 拿 key。 在DaoXE 控制台注册并创建一个 API key。
  2. 查你的模型 ID。 模型 ID 按账户区分 —— 调用 models 端点复制准确 ID，别照抄博客里的字符串。
  3. 把客户端指向 base URL。 base URL 填 https://daoxe.com/v1，填 key 和准确模型 ID。Claude Code 走原生 Messages 路径。
bash
export DAOXE_API_KEY="你的key"

curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

三种协议，一个端点#

协议端点适用

OpenAI ResponsesPOST /v1/responses使用 Responses API 的项目

同一主机、同一 key —— 按你客户端说的协议选。

别听我们说 —— 拿 benchmark 验#

「又快又稳」是说法；能重跑的数字才是证据。配套项目 llm-gateway-benchmark 会测连通率、p50/p95 延迟和每百万 token 价格，且从不存储你的 key、prompt 与响应。跑法见网关基准教程。

可验证才是关键

把 CLI 对准 DaoXE、官方 API 和你现在的供应商，然后对比。如果一家在你要 benchmark 时躲闪 —— 那就是答案。学习如何鉴别掉包/降级。

常见问题#

DaoXE 是 OpenAI 兼容的吗？

是。base URL 填 https://daoxe.com/v1 并用你的 key；现有 OpenAI SDK 代码无需改动即可运行。同时提供 OpenAI Responses 与原生 Anthropic Messages。

Claude Code 能通过 DaoXE 跑吗？

能 —— 在 /v1/messages 原生支持 Anthropic Messages 协议，工具调用与流式都正常。见Cursor / Claude Code 教程。

该用哪个模型 ID？

始终用你账户 GET /v1/models 返回的准确 ID —— ID 按账户区分，别照抄别人的字符串。

怎么确认拿到的是真模型？

用开源、可复现的 benchmark 对准 DaoXE 和官方 API 做对比。见如何鉴别掉包/降级。

有免费试用额度吗？
