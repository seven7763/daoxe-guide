在 Aider 里用 DaoXE

Aider 是终端里懂 git 的结对程序员。设两个环境变量、给模型加 openai/ 前缀，即可跑在 DaoXE 上 —— 一个 key 调 GPT、Claude、Gemini、DeepSeek 等。

更新于 2026-07-20

Aider 通过 OPENAI_API_BASE 和 OPENAI_API_KEY 访问 OpenAI 兼容端点。把 base 指向 DaoXE，并给模型加 openai/ 前缀，Aider 就会走 OpenAI 兼容路径。便宜的 --weak-model 写提交信息，强模型做编辑 —— 见最便宜 API 教程。

为什么让 Aider 走 DaoXE#

  • 终端 + 原生 git。 Aider 自己提交 diff；DaoXE 只负责给模型。
  • 强弱分工。 顶级模型做编辑、便宜模型写提交信息，一个 key。
  • 无锁定。 两个环境变量；随时可撤。

在 Aider 里配置#

  1. 导出端点。 设 OPENAI_API_BASE=https://daoxe.com/v1，OPENAI_API_KEY = 你的 DaoXE key。
  2. 用 openai/ 模型启动。 aider --model openai/YOUR_EXACT_MODEL_ID —— openai/ 前缀必需。
  3. 加 weak 模型（可选）。 --weak-model openai/YOUR_SMALL_MODEL_ID 用于便宜的提交信息与摘要。
  4. 持久化（可选）。 把变量写进 .env，或把模型写进 .aider.conf.yml，免得每次重输。
bash
export OPENAI_API_BASE="https://daoxe.com/v1"
export OPENAI_API_KEY="your_api_key"

# The openai/ prefix tells Aider to use the OpenAI-compatible path
aider --model openai/YOUR_EXACT_MODEL_ID

# optional: a cheaper model for commit messages / summaries
#   --weak-model openai/YOUR_SMALL_MODEL_ID

值得注意的坑

openai/ 前缀很关键：Aider 底层用 LiteLLM，前缀决定走向你 OPENAI_API_BASE 的 OpenAI 兼容路由。用 GET /v1/models 里的准确 ID。部分推理模型需要额外参数，见 Aider 的模型设置。别在 shell 里另设一个会冲突的真实 OPENAI_API_KEY。

验证你拿到的是真模型#

先证明端点可用，再怀疑客户端 —— 这一步失败，改任何设置都没用：

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

先单独验证端点，再在 temperature 0 下把固定 prompt 与官方 API 对比，然后再让它做大重构：

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

常见问题#

为什么模型要加 openai/ 前缀？

Aider 用 LiteLLM；openai/ 告诉它走通往你 OPENAI_API_BASE（DaoXE）的 OpenAI 兼容路由。

base URL 在哪设？

导出 OPENAI_API_BASE=https://daoxe.com/v1 和 OPENAI_API_KEY（DaoXE key）。

能用便宜模型写提交信息吗？

能 —— 传 --weak-model openai/<small-id>；Aider 用它写提交信息和摘要。

哪些模型 ID 可用？

你账号 GET /v1/models 里的任意准确 ID，加 openai/ 前缀。
