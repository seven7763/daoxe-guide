在 Cursor 与 Claude Code 里用更便宜的 endpoint

改一个 base URL 和 key，照常干活。这里给出 Cursor（OpenAI 兼容）和 Claude Code（原生 Anthropic Messages）的精确配置、没人明说的注意事项，以及一步验证。

更新于 2026-07-20

Cursor（OpenAI 兼容）#

  1. 打开 Cursor Settings → Models。 在 API Keys 下启用 OpenAI API Key 并粘贴你的 DaoXE key。
  2. 覆盖 base URL。 打开 Override OpenAI Base URL，填 https://daoxe.com/v1。
  3. 添加准确模型 ID。 在 Model Names 里添加来自 GET /v1/models 的准确 ID。猜的名字会静默失败。
  4. Verify 并测试。 点 Verify（会发一个实时请求），再用一条普通对话测试。

值得知道的注意事项

即使用了自定义 key，Cursor 仍会把请求经由其自有服务器转发（隐私/延迟考量）。Tab 自动补全和部分 Cursor 专有功能可能仍走 Cursor 内置模型。Models 界面各版本会变 —— 请对照客户端配置说明。

Claude Code（原生 Anthropic Messages）#

Claude Code 说的是 Anthropic Messages，不是 OpenAI chat。DaoXE 在 /v1/messages 原生提供该协议，因此工具调用和流式都正常。只需两个环境变量 —— 无需代理或 router。

bash
# Claude Code 读取主机根，会自己补上 /v1/messages
export ANTHROPIC_BASE_URL="https://daoxe.com"
export ANTHROPIC_AUTH_TOKEN="你的key"   # 部分版本读 ANTHROPIC_API_KEY
# 可选：从 GET /v1/models 取准确 ID 固定模型
export ANTHROPIC_MODEL="准确的模型ID"
export ANTHROPIC_SMALL_FAST_MODEL="准确的小模型ID"

claude

两个坑

1）base URL 是主机根（https://daoxe.com），不是 .../v1 —— Claude Code 会自己补 /v1/messages。2）鉴权头形态：部分版本读 ANTHROPIC_AUTH_TOKEN，另一些读 ANTHROPIC_API_KEY；鉴权失败就两个都设。完整走查见Claude Code 教程。

在配置 Claude Code 前，先用裸请求验证协议 —— 如果这步失败，改客户端设置也没用：

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/messages \
  -H "x-api-key: ${DAOXE_API_KEY}" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "你的准确模型ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "回复 OK"}]
  }'

验证你没被降级#

便宜中转有时会换小模型或截断上下文 —— 通常在几周后。按计划在 temperature=0 下跑一套固定探针，与官方 API 做 diff。见如何鉴别掉包/降级与开源 benchmark。

常见问题#

会不会弄坏我现有的 Cursor 配置？

不会 —— 只是改 base URL 和 key，随时可回退。注意部分 Cursor 功能仍用其内置模型。

为什么 Claude Code 用主机根而不是 /v1？

Claude Code 会自己拼 /v1/messages，所以 ANTHROPIC_BASE_URL 必须是 https://daoxe.com。指到 .../v1 会 404。

用 ANTHROPIC_AUTH_TOKEN 还是 ANTHROPIC_API_KEY？

因 Claude Code 版本而异。若其一无法鉴权，就把两个都设为同一个 key。

能用哪些模型？

你账户 GET /v1/models 暴露的那些 —— GPT、Claude、Gemini、DeepSeek 等。请用准确 ID。
