在你的工具里用 DaoXE

一个 OpenAI 兼容 key，接进你已经在用的工具。跳到你的编辑器、CLI 智能体、桌面客户端或自托管 UI —— 或用决策表挑最短路径。

更新于 2026-07-20

DaoXE 是 OpenAI 兼容的，只有一个 base URL（https://daoxe.com/v1），并为 Claude Code 提供原生 Anthropic Messages，所以多数工具只需改 base URL 和 key。这个索引站汇总了所有接入教程、所有模型家族教程，以及诚实的网关对比。不知从哪开始？用决策表 —— 并始终如何鉴别掉包，别让便宜端点悄悄降级你。

编辑器与 CLI 智能体#

一工具一页 —— 真实、针对该工具的配置与坑，而非套模板：

  • Cursor — Cursor 走 Override OpenAI Base URL + 自定义模型。
  • Cline — Cline 走 OpenAI Compatible 供应商。
  • Roo Code — Roo Code 走 OpenAI Compatible，按模式选模型。
  • Continue — Continue 走 config.yaml 的 apiBase。
  • Aider — Aider 走 OPENAI_API_BASE + openai/ 前缀。
  • Zed — Zed 走 language_models.openai_compatible。
  • opencode — opencode 走 opencode.json 里的自定义供应商。
  • Cherry Studio — Cherry Studio 走 OpenAI 类型供应商。
  • Open WebUI — Open WebUI 走 OpenAI API 连接。

按模型家族#

用你的一个 key 触达特定模型家族 —— 附每个模型真实的注意点：

  • Claude — Claude 走 OpenAI chat + 原生 Anthropic Messages。
  • GPT — GPT 走 chat + Responses；OpenAI SDK 即插即用。
  • Gemini — Gemini 走 /v1/chat/completions；无需 Google 密钥。
  • DeepSeek — DeepSeek 走 /v1/chat/completions；注意 reasoning_content。
  • Kimi (Moonshot) — Kimi 走 /v1/chat/completions；要普通输出就关思考。
  • Qwen — Qwen 走 /v1/chat/completions；注意 enable_thinking。
  • Doubao — Doubao 走 /v1/chat/completions；用目录 ID，免 ep- 配置。

常见问题#

最快怎么开始？

在编辑器列表或决策表里找到你的工具，base URL 填 https://daoxe.com/v1，粘贴 key。多数工具无需更多设置。

哪些工具支持原生 Claude Code？

Claude Code 用原生 Anthropic Messages —— 见 Claude 模型教程与Cursor·Claude Code 教程。把 ANTHROPIC_BASE_URL 指向主机根。

如何确保拿到真模型？

在 temperature 0 下把同一 prompt 与官方 API 对比，并学会如何鉴别掉包。便宜端点只有在给你真模型时才算省。

一个 key 能同时驱动多个工具吗？

能 —— 同一个 DaoXE key 适用于这里的每个接入，共用一个余额。当前优势：多数主流模型经低价分组约低 30%–80%。
