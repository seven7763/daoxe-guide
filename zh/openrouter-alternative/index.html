一个你真能 benchmark 的 OpenRouter 替代

一个 key 调 GPT、Claude、Gemini、DeepSeek 等 —— OpenAI 兼容，Claude Code 走原生 Anthropic Messages，且以「可验证」而非「靠信任」为设计目标。

更新于 2026-07-20

为什么大家找替代#

聚合器很方便，但整个品类有个盲点：很难判断某个 endpoint 给的是你要的模型，还是更小/量化/降级的版本。DaoXE 就是来补这个缺口的 —— 并且原生支持 Claude 协议，这是纯 OpenAI 路由做不到的。

DaoXE 给你什么#

一个 key，多种模型

GPT · Claude · Gemini · DeepSeek · Kimi · Qwen · Doubao 统一在一个 endpoint、一份账单下。

原生 Anthropic Messages

Claude Code 和 Anthropic SDK 走 /v1/messages，不是套一层 OpenAI 壳。

可被 benchmark 验证

把开源 CLI对准 DaoXE 和任何一家做对比。

面向海外

面向出海开发者与俄语开发者。不向中国大陆提供服务。

几分钟迁移#

  1. 换 base URL。 把 OpenAI 兼容客户端指向 https://daoxe.com/v1 并填 DaoXE key。
  2. 用准确模型 ID。 从 GET /v1/models 取 —— 别复用别家的 ID。
  3. 把 Claude Code 切到原生 Messages。 环境变量配置见Cursor / Claude Code 教程。
  4. 两边都 benchmark。 在彻底切换前，对比延迟与模型行为。
python
from openai import OpenAI

client = OpenAI(
    base_url="https://daoxe.com/v1",   # 一个 base URL 调所有模型
    api_key="你的DaoXE_key",
)
resp = client.chat.completions.create(
    model="你的准确模型ID",             # 来自 GET /v1/models
    messages=[{"role": "user", "content": "你好"}],
    max_tokens=64,
)
print(resp.choices[0].message.content)

常见问题#

DaoXE 和其他聚合器有何不同？

它以「可被 benchmark」为设计目标（开源 CLI、账户维度模型列表），并为 Claude Code 原生支持 Anthropic Messages，而非仅套 OpenAI 壳。

它 OpenAI 兼容，我能直接切吗？

能 —— 把 base URL 改成 https://daoxe.com/v1，保留 OpenAI SDK 代码，用你账户里的准确 ID。

支持 Claude Code 吗？

支持，原生走 /v1/messages。见配置教程。

可用性上有什么要注意的？

DaoXE 不向中国大陆提供服务；目标是出海开发者、海外华人与俄语开发者。
