DaoXE в Cline

Cline — автономный агент для VS Code. Направьте его на OpenAI-совместимый эндпоинт DaoXE и управляйте правками файлов и tool use через GPT, Claude, Gemini, DeepSeek и другие — одним ключом.

Обновлено 2026-07-20

Cline работает с любым OpenAI-совместимым эндпоинтом, поэтому переход на DaoXE — это смена base URL и ключа: без прокси и форков. Выгода: один ключ для гайд по дешёвым API по всем вендорам и агент, который планирует на топ-модели, а правит на дешёвой.

Зачем направлять Cline через DaoXE#

  • Один ключ, много моделей. Меняйте модель агента без отдельных аккаунтов, карт и счетов у каждого вендора.
  • Маршрутизация по цене. Дешёвая модель на рутинные правки, топовая — на планирование, всё на одном балансе.
  • Нативный tool calling. Агентные правки и запуск команд работают как надо, если модель поддерживает инструменты.

Настройка в Cline#

  1. Откройте настройки провайдера. В панели Cline нажмите шестерёнку и выберите API Provider = OpenAI Compatible.
  2. Укажите Base URL. Введите https://daoxe.com/v1 — заканчивайте на /v1, не на /chat/completions (Cline сам добавит путь).
  3. Вставьте ключ и Model ID. Добавьте ключ DaoXE и Model ID, скопированный дословно из GET /v1/models. Придуманные имена не сработают.
  4. Нажмите Verify. Cline отправит один живой запрос; после успеха запустите небольшую задачу и проверьте правки и tool use.

Что важно учесть

Base URL должен заканчиваться на /v1 — хвост /chat/completions ломает URL. ID моделей привязаны к аккаунту, берите их из GET /v1/models. Режиму агента нужна модель с настоящим tool-calling; если правки не применяются, смените модель и задайте контекстное окно в Model Configuration. Больше клиентов — в заметки по настройке клиентов.

Убедитесь, что отдают настоящую модель#

Сначала проверьте эндпоинт, а не клиент — если это не работает, настройки не помогут:

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

Дешёвый эндпоинт выгоден, только если отдаёт настоящую модель. Сначала проверьте связь, затем сравните вывод с официальным API при temperature 0:

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

Проверяйте нас, а не верьте на слово

Направьте открытый бенчмарк на DaoXE и на официальный API и сравните при temperature 0. Затем изучите, как поймать подмену модели, чтобы дешёвый эндпоинт не подменил вам модель на меньшую.

Частые вопросы#

Cline работает с DaoXE?

Да — выберите провайдер OpenAI Compatible, укажите base URL https://daoxe.com/v1 и вставьте ключ и точный ID модели.

Почему Model ID «не найден»?

ID должен совпадать со строкой из GET /v1/models для вашего аккаунта. Копируйте дословно; ID привязаны к аккаунту.

Можно Claude и GPT в одной настройке?

Да — добавьте несколько ID и переключайтесь на одном балансе. Текущее преимущество в цене: обычно на ~30–80% ниже прайса (дешёвые группы).

Правки агента не применяются — почему?

Используйте модель с нативным tool calling и задайте достаточное контекстное окно в Model Configuration.
