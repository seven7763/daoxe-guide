Доступ к DeepSeek API, OpenAI-совместимо — и ловушка reasoning_content

DeepSeek дёшев и силён в рассуждениях. Через DaoXE это обычный OpenAI-совместимый вызов — но у thinking-варианта есть реальный многошаговый нюанс, который стоит знать до продакшена.

Обновлено 2026-07-20

DeepSeek — одно из лучших по цене/качеству семейств, и регистрация из-за рубежа часто мучительна — через DaoXE он за тем же ключом, что GPT и Claude. Вызывайте /v1/chat/completions с ID deepseek-*. Как вписать его в дешёвый/топ сплит — гайд по дешёвым API.

Как вызвать DeepSeek через DaoXE#

Используйте /v1/chat/completions с точным ID deepseek-*. Есть варианты без размышлений и с ними; thinking-вариант возвращает отдельное поле reasoning_content.

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'
python
# DeepSeek "thinking" variants return a separate reasoning_content field.
# In multi-turn tool use you MUST echo the previous assistant message back
# (reasoning_content included) or the API answers 400.
from openai import OpenAI

client = OpenAI(base_url="https://daoxe.com/v1", api_key="YOUR_DAOXE_KEY")
r = client.chat.completions.create(
    model="YOUR_EXACT_MODEL_ID",   # a deepseek-* id from GET /v1/models
    messages=[{"role": "user", "content": "2+2? Think briefly, then answer."}],
)
msg = r.choices[0].message
print(getattr(msg, "reasoning_content", None))  # chain-of-thought, if exposed
print(msg.content)                               # final answer

Нюансы именно для DeepSeek#

  • Round-trip reasoning_content. С thinking-вариантом в многошаговом tool use НУЖНО возвращать предыдущее сообщение ассистента (с reasoning_content), иначе API ответит 400.
  • Без размышлений — без round-trip. Если видимые рассуждения не нужны, берите non-thinking вариант и избегайте ловушки.
  • Фреймворки теряют поле. Некоторые клиенты молча выбрасывают reasoning_content — проверьте свой перед включением thinking.
  • Только точные ID. Выбирайте нужный вариант по ID из GET /v1/models.

Убедитесь, что отдают настоящую модель#

Сначала проверьте эндпоинт, а не клиент — если это не работает, настройки не помогут:

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \

Частые вопросы#

Почему 400 на втором шаге?

Thinking-варианты требуют вернуть предыдущее сообщение ассистента с reasoning_content в следующий запрос; без него API отвечает 400.

Как избежать ловушки reasoning_content?

Либо честно возвращайте reasoning_content, либо используйте non-thinking вариант, если видимые рассуждения не нужны.

Можно DeepSeek из-за рубежа без китайского аккаунта?

Да — через DaoXE он за вашим единым ключом, без обычных сложностей регистрации за рубежом.
