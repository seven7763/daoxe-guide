DaoXE в Aider

Aider — парный программист в терминале, знающий git. Задайте две переменные окружения и префикс openai/ у модели, чтобы запустить его на DaoXE — GPT, Claude, Gemini, DeepSeek и другие одним ключом.

Обновлено 2026-07-20

Aider читает OPENAI_API_BASE и OPENAI_API_KEY для OpenAI-совместимых эндпоинтов. Направьте базу на DaoXE и добавьте префикс openai/, чтобы Aider пошёл по OpenAI-совместимому пути. Дешёвая --weak-model — на сообщения коммитов, сильная — на правки; см. гайд по дешёвым API.

Зачем направлять Aider через DaoXE#

  • Терминал + git. Aider сам коммитит диффы; DaoXE лишь даёт модель.
  • Сильная + слабая. Топ-модель на правки, дешёвая на коммиты, один ключ.
  • Без залипания. Две переменные; откат в любой момент.

Настройка в Aider#

  1. Экспортируйте эндпоинт. OPENAI_API_BASE=https://daoxe.com/v1 и OPENAI_API_KEY = ваш ключ DaoXE.
  2. Запуск с моделью openai/. aider --model openai/YOUR_EXACT_MODEL_ID — префикс openai/ обязателен.
  3. Слабая модель (опц.). --weak-model openai/YOUR_SMALL_MODEL_ID для дешёвых коммитов и резюме.
  4. Зафиксируйте (опц.). Переменные в .env или модель в .aider.conf.yml, чтобы не вводить заново.
bash
export OPENAI_API_BASE="https://daoxe.com/v1"
export OPENAI_API_KEY="your_api_key"

# The openai/ prefix tells Aider to use the OpenAI-compatible path
aider --model openai/YOUR_EXACT_MODEL_ID

# optional: a cheaper model for commit messages / summaries
#   --weak-model openai/YOUR_SMALL_MODEL_ID

Что важно учесть

Префикс openai/ важен: под капотом Aider использует LiteLLM, и префикс выбирает OpenAI-совместимый маршрут к вашему OPENAI_API_BASE. Берите точные ID из GET /v1/models. Некоторым reasoning-моделям нужны доп. флаги — см. настройки моделей Aider. Не держите конфликтующий реальный OPENAI_API_KEY в шелле.

Убедитесь, что отдают настоящую модель#

Сначала проверьте эндпоинт, а не клиент — если это не работает, настройки не помогут:

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

Проверьте эндпоинт отдельно, затем сравните фиксированный промпт с официальным API при temperature 0, прежде чем доверять крупные рефакторинги:

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

Проверяйте нас, а не верьте на слово

Направьте открытый бенчмарк на DaoXE и на официальный API и сравните при temperature 0. Затем изучите, как поймать подмену модели, чтобы дешёвый эндпоинт не подменил вам модель на меньшую.

Частые вопросы#

Зачем префикс openai/?

Aider использует LiteLLM; openai/ указывает на OpenAI-совместимый маршрут к вашему OPENAI_API_BASE (DaoXE).

Где задать base URL?

Экспортируйте OPENAI_API_BASE=https://daoxe.com/v1 и OPENAI_API_KEY (ключ DaoXE).

Дешёвая модель на коммиты?

Да — --weak-model openai/<small-id>; Aider использует её для сообщений коммитов и резюме.

Какие ID моделей подойдут?

Любой точный ID из GET /v1/models вашего аккаунта, с префиксом openai/.
