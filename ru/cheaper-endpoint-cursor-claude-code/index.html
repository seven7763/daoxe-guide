Дешёвый endpoint в Cursor и Claude Code

Поменяйте base URL и ключ — и продолжайте работать. Вот точная настройка Cursor (OpenAI-совместимо) и Claude Code (нативный Anthropic Messages), оговорки, о которых молчат, и шаг проверки.

Обновлено 2026-07-20

Cursor (OpenAI-совместимо)#

  1. Откройте Cursor Settings → Models. В API Keys включите OpenAI API Key и вставьте ключ DaoXE.
  2. Переопределите base URL. Включите Override OpenAI Base URL и укажите https://daoxe.com/v1.
  3. Добавьте точные ID моделей. В Model Names добавьте точные ID из GET /v1/models. Угаданные имена молча не работают.
  4. Verify и тест. Нажмите Verify (отправит один живой запрос), затем проверьте обычным сообщением в чате.

Важные оговорки

Cursor всё равно проксирует запросы через свои серверы даже с вашим ключом (вопрос приватности/задержки). Tab-автодополнение и часть функций могут по-прежнему использовать встроенные модели Cursor. UI Models меняется между версиями — сверяйтесь с заметками по настройке клиентов.

Claude Code (нативный Anthropic Messages)#

Claude Code говорит на Anthropic Messages, а не на OpenAI chat. DaoXE отдаёт этот протокол нативно на /v1/messages, поэтому вызов инструментов и стриминг работают. Достаточно двух переменных окружения — без прокси и роутера.

bash
# Claude Code берёт КОРЕНЬ хоста и сам добавляет /v1/messages
export ANTHROPIC_BASE_URL="https://daoxe.com"
export ANTHROPIC_AUTH_TOKEN="ваш_ключ"   # в части версий читается ANTHROPIC_API_KEY
# опционально: точные ID из GET /v1/models
export ANTHROPIC_MODEL="точный_id_модели"
export ANTHROPIC_SMALL_FAST_MODEL="точный_id_быстрой_модели"

claude

Две ловушки

1) Base URL — это корень хоста (https://daoxe.com), а не .../v1: путь /v1/messages Claude Code добавит сам. 2) Форма заголовка авторизации: часть версий читает ANTHROPIC_AUTH_TOKEN, другие — ANTHROPIC_API_KEY; при ошибке экспортируйте оба. Полный разбор: гайд по Claude Code.

Проверьте протокол сырым запросом до настройки Claude Code — если он падает, правка настроек клиента не поможет:

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/messages \
  -H "x-api-key: ${DAOXE_API_KEY}" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "ТОЧНЫЙ_ID_МОДЕЛИ",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Ответь OK."}]
  }'

Проверьте, что вас не даунгрейдят#

Дешёвые релеи иногда отдают меньшую модель или режут контекст — обычно спустя недели. Гоняйте фиксированный набор проб при temperature=0 и сравнивайте с официальным API по расписанию. См. как распознать подмену модели и открытый бенчмарк.

Частые вопросы#

Сломается ли мой текущий Cursor?

Нет — это смена base URL и ключа, откатывается в любой момент. Учтите, что часть функций Cursor всё равно использует встроенные модели.

Почему Claude Code использует корень хоста, а не /v1?

Claude Code сам строит путь /v1/messages, поэтому ANTHROPIC_BASE_URL должен быть https://daoxe.com. Указание .../v1 даёт 404.

ANTHROPIC_AUTH_TOKEN или ANTHROPIC_API_KEY?

Зависит от версии Claude Code. Если одно не авторизует — экспортируйте обе переменные на тот же ключ.

Какие модели доступны?

Те, что отдаёт ваш аккаунт в GET /v1/models — GPT, Claude, Gemini, DeepSeek и другие. Используйте точные ID.
