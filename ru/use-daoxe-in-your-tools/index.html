DaoXE в ваших инструментах

Один OpenAI-совместимый ключ, встроенный в инструменты, которыми вы уже пользуетесь. Перейдите к своему редактору, CLI-агенту, десктоп-клиенту или self-hosted UI — или используйте таблицу выбора.

Обновлено 2026-07-20

DaoXE OpenAI-совместим, с единым base URL (https://daoxe.com/v1) и нативным Anthropic Messages для Claude Code, поэтому большинству инструментов нужна лишь смена base URL и ключа. Этот хаб связывает все гайды по интеграциям, по семействам моделей и честные сравнения шлюзов. Не знаете, с чего начать? Используйте таблицу выбора — и всегда как поймать подмену модели, чтобы дешёвый эндпоинт не понизил вас незаметно.

Редакторы и CLI-агенты#

По странице на инструмент — реальная настройка и нюансы, а не copy-paste шаблон:

  • Cursor — Cursor через Override OpenAI Base URL + кастомные модели.
  • Cline — Cline через провайдер OpenAI Compatible.
  • Roo Code — Roo Code через OpenAI Compatible, модели по режимам.
  • Continue — Continue через apiBase в config.yaml.
  • Aider — Aider через OPENAI_API_BASE + префикс openai/.
  • Zed — Zed через language_models.openai_compatible.
  • opencode — opencode через кастомный провайдер в opencode.json.
  • Cherry Studio — Cherry Studio через провайдер типа OpenAI.
  • Open WebUI — Open WebUI через подключение OpenAI API.

По семействам моделей#

Доступ к конкретному семейству моделей одним ключом — с реальными нюансами каждой:

  • Claude — Claude через OpenAI chat + нативный Anthropic Messages.
  • GPT — GPT через chat + Responses; drop-in OpenAI SDK.
  • Gemini — Gemini через /v1/chat/completions; ключ Google не нужен.
  • DeepSeek — DeepSeek через /v1/chat/completions; помните про reasoning_content.
  • Kimi (Moonshot) — Kimi через /v1/chat/completions; отключайте thinking для обычного вывода.
  • Qwen — Qwen через /v1/chat/completions; помните про enable_thinking.
  • Doubao — Doubao через /v1/chat/completions; каталожный ID, без ep-.

Частые вопросы#

Как начать быстрее всего?

Найдите свой инструмент в списке редакторов или в таблице выбора, задайте base URL https://daoxe.com/v1 и вставьте ключ. Большинству инструментов больше ничего не нужно.

Какие инструменты поддерживают нативный Claude Code?

Claude Code использует нативный Anthropic Messages — см. гайд по Claude и гайд Cursor и Claude Code. Укажите ANTHROPIC_BASE_URL на корень хоста.

Как убедиться, что отдают настоящую модель?

Сравните один и тот же промпт с официальным API при temperature 0 и освойте, как поймать подмену модели. Дешёвый эндпоинт выгоден, только если отдаёт настоящую модель.

Один ключ может обслуживать несколько инструментов сразу?

Да — один и тот же ключ DaoXE работает во всех интеграциях здесь, на одном балансе. Текущее преимущество: обычно на ~30–80% ниже прайса (дешёвые группы).
