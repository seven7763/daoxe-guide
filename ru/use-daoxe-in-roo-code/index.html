DaoXE в Roo Code

Roo Code — агент для VS Code (форк Cline), принимающий любой OpenAI-совместимый провайдер. Направьте его на DaoXE для одного ключа к GPT, Claude, Gemini, DeepSeek и другим — при желании со своей моделью на каждый режим.

Обновлено 2026-07-20

Roo Code принимает любой OpenAI-совместимый эндпоинт, поэтому переход на DaoXE — смена base URL и ключа. Особенность Roo — профили моделей по режимам: топ-модель в Architect и дешевле в Code, всё на одном ключе. Где реально экономия — см. гайд по дешёвым API.

Зачем направлять Roo Code через DaoXE#

  • Модели по режимам. Разные модели DaoXE для Code, Ask и Architect.
  • Один ключ, много вендоров. Без отдельных аккаунтов для GPT, Claude, Gemini, DeepSeek или Kimi.
  • Drop-in. Если уже используете Cline — настройки те же, Roo это форк.

Настройка в Roo Code#

  1. Откройте настройки Roo. Шестерёнка в панели Roo Code, затем в Providers выберите OpenAI Compatible.
  2. Base URL и ключ. Base URL https://daoxe.com/v1 (до /v1, без /chat/completions), вставьте ключ DaoXE.
  3. Выберите Model ID. Укажите точный ID из GET /v1/models; Roo может подтянуть список по вашему ключу.
  4. Назначьте по режимам (опц.). Дайте Code, Ask и Architect разные модели для баланса цены и качества.

Что важно учесть

Base URL не должен включать /chat/completions — Roo добавит путь сам. Действиям агента нужен нативный tool calling; выберите модель с поддержкой инструментов, иначе правки не сработают. ID моделей привязаны к аккаунту (см. GET /v1/models). Держите отдельные профили провайдера, если переключаетесь. Ещё клиенты: заметки по настройке клиентов.

Убедитесь, что отдают настоящую модель#

Сначала проверьте эндпоинт, а не клиент — если это не работает, настройки не помогут:

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

Убедитесь, что эндпоинт отвечает, затем сравните сложный промпт с официальным API при temperature 0, чтобы исключить даунгрейд:

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

Проверяйте нас, а не верьте на слово

Направьте открытый бенчмарк на DaoXE и на официальный API и сравните при temperature 0. Затем изучите, как поймать подмену модели, чтобы дешёвый эндпоинт не подменил вам модель на меньшую.

Частые вопросы#

Настройка Roo Code как в Cline?

В основном да — Roo это форк Cline. Выберите OpenAI Compatible, укажите https://daoxe.com/v1, вставьте ключ и точный ID модели.

Разная модель на каждый режим?

Да — Roo поддерживает профили по режимам: Architect на топ-модели, Code на дешёвой.

Вызовы инструментов падают?

Roo нужен нативный tool calling. Смените ID на модель с поддержкой инструментов.

Где взять ID моделей?

Вызовите GET /v1/models с ключом; копируйте точные ID (они привязаны к аккаунту).

Попробуйте DaoXE — и проверьте его сами
