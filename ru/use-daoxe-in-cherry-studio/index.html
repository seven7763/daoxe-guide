DaoXE в Cherry Studio

Cherry Studio — кроссплатформенный десктоп-клиент для множества провайдеров. Добавьте DaoXE как провайдер типа OpenAI и получите GPT, Claude, Gemini, DeepSeek и другие одним ключом — если правильно задать путь в API host.

Обновлено 2026-07-20

Cherry Studio воспринимает DaoXE как провайдер типа OpenAI: вставьте ключ, задайте API host и добавьте ID моделей. Единственное, на чём спотыкаются, — как Cherry достраивает путь эндпоинта. Сделайте это правильно — и всё заработает. Деление дешёвых и топ-моделей — в гайд по дешёвым API.

Зачем направлять Cherry Studio через DaoXE#

  • Локальный десктоп, много провайдеров. Чаты остаются локально, а модели даёт DaoXE.
  • Один ключ, много моделей. Добавьте ID GPT, Claude, Gemini и DeepSeek под одним провайдером DaoXE.
  • Модель на ассистента. Назначайте разным ассистентам разные модели DaoXE для баланса цены и качества.

Настройка в Cherry Studio#

  1. Добавьте провайдер. Settings → Model Providers → Add, тип OpenAI, имя DaoXE.
  2. Ключ и API host. Вставьте ключ DaoXE. В host укажите https://daoxe.com и дайте Cherry достроить путь (точное правило — в нюансах).
  3. Добавьте модели. Загрузите или добавьте точные ID из GET /v1/models (например, ID GPT, Claude, DeepSeek).
  4. Выберите модель и проверьте. Выберите модель DaoXE в чате и отправьте сообщение, чтобы убедиться в ответе.

Что важно учесть

Классический нюанс — автодополнение пути в API host у Cherry. Надёжно: укажите https://daoxe.com и дайте Cherry добавить /v1/chat/completions; либо введите полный эндпоинт с # в конце, чтобы принять его дословно. Host с / на конце обрабатывается иначе — если список моделей не грузится или сыпятся 404, дело почти всегда в этом. Берите точные ID из GET /v1/models; добавьте вручную, если автозагрузка выдаёт другой набор. Ключи хранятся в локальном приложении.

Убедитесь, что отдают настоящую модель#

Сначала проверьте эндпоинт, а не клиент — если это не работает, настройки не помогут:

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

Сначала проверьте эндпоинт вне Cherry, затем сравните фиксированный промпт с официальным API при temperature 0:

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

Проверяйте нас, а не верьте на слово

Направьте открытый бенчмарк на DaoXE и на официальный API и сравните при temperature 0. Затем изучите, как поймать подмену модели, чтобы дешёвый эндпоинт не подменил вам модель на меньшую.

Частые вопросы#

Какой тип провайдера выбрать?

Выберите OpenAI — DaoXE OpenAI-совместим. Задайте ключ и API host, затем добавьте ID моделей.

Что писать в API host?

Укажите https://daoxe.com и дайте Cherry добавить /v1/chat/completions, либо введите полный путь с # в конце, чтобы принять его дословно.

Модели не грузятся или 404 — почему?

Почти всегда правило пути в API host. Перепроверьте host по нюансам и проверьте ключ через GET /v1/models.

Разные ассистенты — разные модели?

Да — модель DaoXE на каждого ассистента, всё на одном ключе. Текущее преимущество в цене: см. страницу цен.
