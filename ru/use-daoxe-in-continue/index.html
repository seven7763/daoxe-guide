DaoXE в Continue

Continue — open-source ассистент в IDE, настраиваемый одним YAML-файлом. Добавьте DaoXE как провайдер openai с кастомным apiBase и назначьте роли — один ключ на chat, edit и apply.

Обновлено 2026-07-20

Всё поведение Continue задаётся в ~/.continue/config.yaml. Добавьте модель с provider: openai и apiBase на DaoXE — и chat/edit/apply пойдут через выбранную модель. Как делить дешёвые и топ-модели по ролям — см. гайд по дешёвым API.

Зачем направлять Continue через DaoXE#

  • Один файл — полный контроль. Модели, роли и провайдеры в config.yaml.
  • Роли распределяют работу. Дешёвая модель DaoXE на autocomplete, топовая на chat/edit.
  • Один ключ, много вендоров. Без аккаунтов у каждого из GPT, Claude, Gemini, DeepSeek или Kimi.

Настройка в Continue#

  1. Откройте config.yaml. Правьте ~/.continue/config.yaml (глобально) или .continue/config.yaml (в репозитории).
  2. Добавьте модель DaoXE. Укажите provider: openai, apiBase: https://daoxe.com/v1, ключ и точный model ID.
  3. Назначьте роли. Добавьте roles: [chat, edit, apply] (и autocomplete на маленькой модели).
  4. Reload config. Нажмите Reload config в расширении и проверьте chat и edit.
yaml
# ~/.continue/config.yaml  (global)  — or .continue/config.yaml per repo
name: DaoXE
version: 0.0.1
schema: v1

models:
  - name: DaoXE - your model
    provider: openai
    model: YOUR_EXACT_MODEL_ID     # from GET /v1/models
    apiBase: https://daoxe.com/v1
    apiKey: YOUR_DAOXE_KEY
    roles: [chat, edit, apply]

Что важно учесть

Используйте YAML — старый config.json ещё грузится, но устарел. Для gpt-5/o-серии Continue по умолчанию идёт на /responses; при ошибках "organization must be verified" или стриминга добавьте useResponsesApi: false, чтобы принудительно использовать /chat/completions. apiBase должен включать /v1. Роли задают, какая модель делает что.

Убедитесь, что отдают настоящую модель#

Сначала проверьте эндпоинт, а не клиент — если это не работает, настройки не помогут:

bash
export DAOXE_API_KEY="your_api_key"

# List the exact model IDs your account can call
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/models \
  -H "Authorization: Bearer ${DAOXE_API_KEY}"

После перезагрузки независимо проверьте эндпоинт, затем сравните фиксированный промпт с официальным API при temperature 0:

bash
curl --fail-with-body --show-error --silent \
  https://daoxe.com/v1/chat/completions \
  -H "Authorization: Bearer ${DAOXE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_EXACT_MODEL_ID",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one sentence."}]
  }'

Проверяйте нас, а не верьте на слово

Направьте открытый бенчмарк на DaoXE и на официальный API и сравните при temperature 0. Затем изучите, как поймать подмену модели, чтобы дешёвый эндпоинт не подменил вам модель на меньшую.

Частые вопросы#

Какой провайдер выбрать для DaoXE?

Используйте provider: openai и apiBase: https://daoxe.com/v1 — DaoXE OpenAI-совместим.

config.yaml или config.json?

YAML. Старый JSON ещё грузится, но устарел; новые опции — в YAML.

Reasoning-модели дают ошибки?

Continue может идти на /responses для gpt-5/o-серии. Поставьте useResponsesApi: false для /chat/completions.

Можно одна модель на автокомплит, другая на чат?

Да — для этого и нужны roles. Дешёвая на autocomplete, сильная на chat/edit.
